# Промпт — скопировать ЦЕЛИКОМ в Claude (запущенный в папке сайта)

---

Ты вносишь правки на сайт частного риэлтора Владимира Белолипцева (премиум-вторичка ЦАО Москвы), собранные из видео-созвона с клиентом. Работаешь точно по ТЗ.

**Где ТЗ и материалы:** папка `TZ-Interview-2026-06-23` (этот же набор файлов, рядом с этим промптом). Главное:
- `01-TZ-MASTER.md` — все 57 правок: сводка по приоритету + по экранам. **Прочитай его целиком первым делом.**
- `screenshots/<ID>.jpg` — кадр экрана из записи на момент реплики клиента. **Перед каждой правкой открой её кадр** и сверь, что нашёл то самое место.
- `04-QA-REPORT.md` — какие правки где находятся (охват по страницам) и где замена «наивная», а где нужна аккуратность по падежам.

**Целевой кодабейз:** `01-Site/Белолипцев (2)/Белолипцев/astro/` (статика на Astro; страницы = `src/data/*.html`; сборка `npx astro build`; прод `astro-seven-delta.vercel.app`).
⚠️ В проекте есть вторая папка `01-Site-Astro/` — она ОТВЯЗАНА от Vercel, из неё НЕ деплоить (перебьёт прод старой версией). Деплой только из целевой папки и только после подтверждения у Ника/владельца.

**Порядок работы:**
1. Составь TodoWrite по всем 🔴 **P0** (контент/SEO/баги) и иди по ним первыми. Затем 🟡 **P1** (визуал/вёрстка/UX).
2. 🟢 **P2 не трогай** без отдельного решения (там сперва сравнивают SEO-частотность: «дореволюционка», «частный/индивидуальный риэлтор», «23 года / 20+ лет», «брокер»).
3. **Сквозные правки** (раздел GLOBAL: G01–G11, P01) применяй на ВСЕХ страницах разом и проверяй грепом, что нигде не осталось старого: `2003→2004`, `Имя→Фамилия впереди`, `₽→рублей` (склонение!), `Топ-4→Топ-3`, контакты в подвал, чекбокс по умолчанию, единый хедер.
4. Замены с падежами/регистром (₽→рублей, коллегия→ГИЛЬДИЯ, имя↔фамилия) делай по каждому вхождению, НЕ слепым replace — иначе поломаешь грамматику.

**Правила (жёстко):**
- Перед каждой нетривиальной правкой прочитай нужный файл сам (не делегируй Explore-агентам — они галлюцинируют ссылки), проследи механику, потом точечный дифф.
- **Бэкап перед правкой; показывай дифф перед применением.** Работай с бэкапами.
- SEO-копию не разрушай — текст-узлы не теряем (текста должно остаться не меньше, чем было).
- Не выдумывай факты: на страницах домов часть текста сгенерена ИИ и врёт (напр. «четырёхуровневые пентхаусы 300+ м²» — их нет). В ТЗ такие места помечены «факт-чек» — при сомнении оставь вопрос клиенту, не сочиняй.

**Не закрывать (ждут материала клиента):** сертификаты в 4 блока (H02), список смежных секторов (U08), 3 рукописных юр-момента. Помечай как «ждёт материала».

**Definition of Done (по каждой правке):**
- Визуально проверено в браузере (скрин/зум секции) — совпадает с кадром из ТЗ.
- `astro build` зелёный; golden-path (главная/услуги/сталинки/подвал) рендерится без регрессий.
- Сквозные правки реально применены на всех страницах (проверено грепом, не «на глаз»).

Начни: прочитай `01-TZ-MASTER.md`, собери TodoWrite по P0, приступай.
